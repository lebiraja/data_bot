# Data Bot Architecture

This document describes the architecture of the **Data Bot** project in a clear, modular, and beginner-friendly way.

---

## Overview

**Data Bot** is a modular Telegram bot that provides two primary features:

1. **Data Cleaning**: Users upload CSV files, which are cleaned and summarized using both standard Python libraries (pandas) and an AI model (Ollama).
2. **AI Chat**: Users can chat with the bot and get answers generated by a local AI model.

## Component Responsibilities

### 1. `bot.py` (Main Controller)
- Initializes the Telegram bot and loads settings.
- Sets up directories and logging.
- Handles Telegram commands and file uploads.
- Routes user requests to the correct module:
  - Chat messages → `chat_handler.py`
  - CSV uploads → `data_cleaner.py`

### 2. `chat_handler.py` (Chat Logic)
- Manages user chat mode and conversation context.
- Sends/receives messages to/from the AI model via `ollama_handler.py`.
- Saves and fetches chat history for each user from the database (`db_handler.py`).

### 3. `data_cleaner.py` (Data Cleaning Logic)
- Loads, analyzes, and cleans CSV files using pandas.
- Generates statistical summaries and cleaning steps.
- Queries `ollama_handler.py` for AI-powered analysis and cleaning advice.
- Returns cleaned files and a human-readable summary.

### 4. `ollama_handler.py` (AI Model Interface)
- Checks if Ollama (local AI language model) is running (API or CLI).
- Sends prompts and receives responses for both chat and data cleaning analysis.
- Handles retries, errors, and switching between API/CLI methods.

### 5. `db_handler.py` (Database Layer)
- Uses SQLite to persist:
  - User profiles and preferences
  - Chat history (for context)
- Provides methods for adding/updating users, managing chat modes, storing/retrieving messages, and clearing history.

### 6. `utils.py` (Utilities)
- Centralized logging configuration (to file and console).
- Cleans up old files in uploads/outputs folders.
- Environment validation (checks for tokens, dependencies, directories).
- Miscellaneous helpers (creating `.env` files, safe file deletion).

## Data Flow

### A. Data Cleaning Request

1. **User uploads a CSV file** in Telegram (Data Cleaning Mode).
2. `bot.py` receives the file, saves it to disk, and calls `data_cleaner.py`.
3. `data_cleaner.py` analyzes and cleans the CSV, asks `ollama_handler.py` for AI advice, and saves the cleaned file.
4. The cleaned file and a summary are sent back to the user by `bot.py`.
5. Temporary files are cleaned up as needed.

### B. Chat Request

1. **User sends a text message** in Telegram (Chat Mode).
2. `bot.py` routes the message to `chat_handler.py`.
3. `chat_handler.py` fetches conversation history from `db_handler.py`, builds a prompt, and calls `ollama_handler.py`.
4. The AI’s reply is saved for context and sent back to the user.
5. User preferences and history are kept in SQLite for personalized interactions.

## Deployment & Environment

- **Runs locally on Python 3.8+**
- **Requires Ollama** (local AI model runner)
- **Uses SQLite** for persistent storage (no external database needed)
- **Configuration** via `.env` file (for Telegram bot token, log level, etc.)
- **Logs** to both file (`data_bot.log`) and console

---

## Error Handling & Logging

- All modules use a centralized logger for info, warnings, and errors.
- User-friendly error messages are sent to users.
- Unexpected errors are logged for debugging.

---

## Extensibility

- To add new features (e.g., support for other file types or more AI models), add new modules or extend existing ones.
- The modular structure makes it easy to test, maintain, and expand.
